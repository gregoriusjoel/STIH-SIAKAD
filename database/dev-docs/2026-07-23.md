  - Penyelarasan tampilan baru ini secara konsisten di seluruh halaman otentikasi mahasiswa:

    - Halaman Atur Ulang Kata Sandi (`reset_password.blade.php`)

  - **Portal Dosen:** Menggunakan ilustrasi kustom [`Teacher-cuate.svg`](file:///Users/joelnaradata/Project/Universitas%20Adhyaksa%20-%20SATU/public/images/Teacher-cuate.svg) dengan 4 widget interaktif melayang yang relevan (Jadwal Mengajar, Absensi Kelas, Bimbingan PA, Input Nilai) serta watermark `PORTAL DOSEN`.
  - **Portal Orang Tua:** Menggunakan ilustrasi kustom [`Back to school-cuate.svg`](file:///Users/joelnaradata/Project/Universitas%20Adhyaksa%20-%20SATU/public/images/Back%20to%20school-cuate.svg) dengan 4 widget interaktif melayang yang relevan (Hasil Studi, Kehadiran Anak, Tagihan Kuliah, Pengumuman) serta watermark `PORTAL ORANG TUA`.

• **[Optimasi Visual] Penyesuaian Dimensi Form Login (Compact Layout)**
  - Mengurangi padding kontainer, margin bawah logo, header, dan grup input di semua portal login agar tampilan formulir lebih padat, ringkas, dan proporsional.
  - Menurunkan tinggi kolom input (`input-field`) dan tombol masuk (`submit-btn`) dari `56px` menjadi `48px`.
  - Menyesuaikan ukuran font judul selamat datang menjadi `28px` agar lebih ramah bagi berbagai resolusi layar.

• **[Perbaikan Tampilan] Penguncian Scrollbar Halaman Utama & Area Kosong Bawah**
  - Menghentikan munculnya scrollbar vertikal browser dengan mengunci properti `height: 100vh` and `overflow: hidden` pada `body` dan kontainer utama `.split-container` di seluruh portal login.
  - Memastikan panel kanan berwarna merah muda-maroon selalu menutup 100% tinggi layar dan tidak menyisakan area kosong di bawahnya akibat pendaran orbs (`.feature-orb-2`) yang melewati batas bawah viewport.
  - Menyematkan properti `overflow-y: auto` pada sisi kolom form `.form-side` sebagai fallback keamanan agar form login tetap dapat di-scroll secara internal pada perangkat seluler beresolusi sangat rendah.

• **[Fitur Baru] Login Menggunakan Username secara Universal & Opsional**
  - Pembuatan migrasi database untuk menambahkan kolom `username` (nullable & unique) di tabel `users`.
  - Penyelarasan logika login pada `LoginController.php` untuk mencocokkan input form login (baik email utama, email mahasiswa, maupun username) secara dinamis.
  - Penyediaan input pengelolaan username pada halaman profil Mahasiswa (`UpdateProfilRequest.php`, `ProfilController.php`, `manajemen.blade.php`) dan profil Dosen (`ProfilController.php`, `edit.blade.php`) lengkap dengan filter auto-sanitasi instan (lowercase dan pembuangan spasi/simbol).
  - Penyediaan input username pada manajemen orang tua oleh Admin (`ParentController.php`, `create.blade.php`, `edit.blade.php`) dan daftar data parent (`index.blade.php`).
  - Penyediaan input username pada manajemen akun umum/admin oleh Admin (`UserController.php`, `create.blade.php`, `edit.blade.php`, `index.blade.php`).
  - Penyesuaian seluruh tipe input form login (Mahasiswa, Dosen, Orang Tua, Admin Utama) dari `email` ke `text` agar mendukung input string username.

• **[Fitur Keamanan] Pemblokiran Copy/Paste Input Kata Sandi**
  - Pencegahan penyalinan (*copy*), pemotongan (*cut*), dan penempelan (*paste*) teks pada kolom input password saat disembunyikan (tipe `password`) di semua form login dan reset password untuk meminimalkan risiko pencurian kata sandi secara fisik/lokal.
