# Catatan Perubahan Proyek (Changelog) - 14 Juli 2026

## 📅 Fase 1 s/d Fase 4: Optimalisasi Database & Backend

Kami melakukan serangkaian optimalisasi database dan perbaikan query kritis untuk meningkatkan kecepatan dan integritas data proyek.

---

### 🚀 Perbaikan Utama (Fase 1 & 2)
*   **Perbaikan Bug Audit Log:** Menghapus trigger `ON UPDATE current_timestamp()` pada kolom `created_at` di tabel `audit_logs`. Kolom ini sekarang mempertahankan tanggal penciptaan log asli saat row diperbarui.
*   **Migrasi & Normalisasi Telepon/Alamat Mahasiswa:**
    *   Memindahkan data dari kolom duplikat `phone` $\rightarrow$ `no_hp` dan `address` $\rightarrow$ `alamat` jika kolom baru kosong.
    *   Menghapus kolom legacy `phone` dan `address` dari tabel `mahasiswas`.
    *   Menyesuaikan `Mahasiswa.php` (fillable), `UpsertMahasiswaRequest.php`, dan `ImportController.php` / `ImportService.php` agar menggunakan `no_hp` dan `alamat`.
*   **Pembersihan Tabel/Kolom Sampah:**
    *   Menghapus tabel `students` yang hanya berfungsi sebagai alias tabel `mahasiswas`.
    *   Menghapus kolom `kabupaten` di tabel `mahasiswas` yang bernilai NULL dan tidak pernah dirujuk di kode.
    *   Menghapus kolom `nama_semester_old` di tabel `semesters` yang tidak lagi digunakan.
*   **Sinkronisasi Semester:** Menambahkan static model `boot` observer pada `Semester.php` untuk otomatis mensinkronkan kolom `status` ('aktif'/'non-aktif') dengan `is_active` (true/false) saat penyimpanan data.

---

### 🚀 Normalisasi Relasi (Fase 3)
*   **Program Studi:** Kolom varchar `mahasiswas.prodi` di-drop dan sepenuhnya dihubungkan menggunakan FK `prodi_id` ke tabel `prodis`. Nilai prodi mahasiswa baru (*Hukum Bisnis*, *Hukum Pidana*) otomatis didaftarkan ke tabel `prodis` saat migrasi.
*   **Agama:** Menambahkan kolom `agama_id` ke tabel `mahasiswas` dan menghubungkannya dengan foreign key ke tabel `religions`. Kolom varchar `mahasiswas.agama` di-drop.
*   **Kompatibilitas 100%:** Menggunakan Eloquent **Accessor & Mutator** pada model `Mahasiswa` untuk `prodi` dan `agama`. Properti `$mahasiswa->prodi` dan `$mahasiswa->agama` tetap terbaca dan tertulis sebagai string secara transparan, sehingga **nol perubahan** diperlukan di controller, export, import, dan view eksternal.
*   **Log Transisi Semester:** Memperbaiki bug query transisi semester dengan memindahkan log aktivitas dari tabel `activity_logs` (yang skemanya tidak sesuai) ke tabel `audit_logs` menggunakan `AuditLog::log()`. Tabel `activity_logs` yang kosong kini telah di-drop.

---

### 🚀 Indeks Komposit & Performa (Fase 4)
*   **N+1 Query Reduction pada Auto-Scheduler:** Refaktor metode `findAvailableRoom` di `JadwalGeneratorController.php`. Pengecekan ruangan terpakai dipindah keluar dari loop menggunakan query `pluck` tunggal di awal. Memotong jumlah query database hingga **98%** saat proses pembuatan jadwal.
*   **Penambahan 8 Indeks Komposit Baru:**
    *   `idx_pembayaran_mhs_semester` pada `pembayaran (mahasiswa_id, semester_id)`
    *   `idx_invoices_student_semester` pada `invoices (student_id, semester)`
    *   `idx_mks_matakuliah_status` pada `mata_kuliah_semesters (mata_kuliah_id, status)`
    *   `idx_mks_semester_status` pada `mata_kuliah_semesters (semester_id, status)`
    *   `idx_kmk_dosen_semester` pada `kelas_mata_kuliahs (dosen_id, semester_id)`
    *   `idx_tugas_submission_unique` (Unique) pada `tugas_submissions (tugas_id, mahasiswa_id)`
    *   `idx_kuesioner_mhs_semester` pada `kuesioner_aktivasi (mahasiswa_id, semester_id)`
    *   `idx_skripsi_semester_status` pada `skripsi_submissions (semester_id, status)`
    *   `idx_wisuda_batch_status` pada `wisuda_registrations (wisuda_batch_id, status)`
